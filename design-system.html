15 / Design Principles
01

Terminal-native

Every element should feel like it belongs in a terminal. Monospace everywhere. Sharp edges. Dense information.

02

One accent only

Amber is the single warm color. It's used for: the cursor, CTAs, highlights, and the pawn ASCII art. Everything else is grayscale.

03

Show, don't tell

The terminal code block IS the hero. Show the deploy command. Show the output. The product sells itself.

04

Brutally minimal

No gradients, no shadows, no rounded corners. Borders are thin and subtle. Whitespace (blackspace) is the primary design tool.

05

Developer credibility

This is a tool for people who write code. The design should signal: "built by engineers, for engineers."
